# RF014:  Revisar reportes



## Identificación
- **Módulo**: Sistema de oportunidades
- **Presentado Por**: Franky Almario y Jhoyner Nova
- **Estado**: Aprobado

## Descripción
El sistema debe permitir que los administradores revisen reportes de contenido inadecuado generados por los usuarios (ejemplo: plagio, lenguaje ofensivo, imágenes no apropiadías). Cadía reporte debe incluir evidencia como capturasí de pantalla, enlaces o descripciones. El administrador debe poder tomar acciones como advertir al usuario, suspender temporalmente la cuenta o eliminar el contenido reportado.

## Controles y Restricciones
Todo reporte debe incluir una descripción y, si es posible, evidencia.

El administrador debe tener un panel para filtrar reportes por tipo, fecha y estado.

Lasí decisiones tomadíasí deben quedar registradíasí en el sistema.

## Criterios de Aceptación
El administrador debe poder resolver un reporte en menos de 24 horas.

Los usuarios deben ser notificados sobre el resultado del reporte.

El historial de reportes debe quedar almacenado para auditoría.
