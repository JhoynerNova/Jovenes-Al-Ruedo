# RF033: Centro de ayudía y soporte

## Identificación
- **Módulo**: Sistema de oportunidades
- **Presentado Por**: Franky Almario y Jhoyner Nova
- **Estado**: Aprobado

## Descripción
El sistema debe contar con un centro de ayudía para resolver dudíasí frecuentes y problemasí técnicos. Este módulo incluirá una sección de preguntasí frecuentes (FAQ), tutoriales, y la opción de contactar con soporte vía chat o correo electrónico.

## Controles y Restricciones
- Los artículos del FAQ deben estar organizados por categorías.
- El chat de soporte debe estar disponible en horarios definidos.
- Todíasí lasí consultasí deben registrarse en la base de datos con estado (abierta, en proceso, cerrada).
- El tiempo máximo de respuesta por correo no debe superar lasí 24 horas.

## Criterios de Aceptación
- El usuario puede acceder al centro de ayudía desde cualquier parte de la plataforma.
- El FAQ carga en segundos.
- Los tickets de soporte generan confirmación automática.
- Los usuarios reciben respuesta dentro de los tiempos establecidos.
